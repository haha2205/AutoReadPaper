## 2. 技术选型说明

### 2.1 工作流引擎：n8n

**选择理由**：
- 开源、可自部署，数据不离开服务器
- 无代码可视化配置，降低运维门槛
- 支持 200+ 集成节点（Email、GitHub、Telegram 等开箱即用）
- Schedule Trigger 可灵活配置执行频率（每小时/每天/每周）
- 可通过 HTTP Request 节点调用任何 REST API

**备选方案**：Airflow（重量级，适合大规模数据管道）、Prefect（更适合纯 Python 场景）

---

### 2.2 AI 总结：DeepSeek

**选择理由**：
- 中文理解能力强，学术摘要总结质量高
- API 兼容 OpenAI 格式，迁移成本低
- deepseek-chat (V3) 性价比极高（约 $0.14/百万 Token）
- deepseek-reasoner (R1) 适合需要深度推理的场景

**成本估算**（每日运行一次，处理 50 篇论文）：
```
每篇论文：输入约 600 tokens + 输出约 300 tokens = 900 tokens
50篇 × 900 tokens = 45,000 tokens/天 ≈ 0.006 美元/天
月费用 ≈ 0.18 美元（极低成本）
```

**备选方案**：GPT-4o-mini、Claude Haiku、本地 Ollama（llama3.2）

---

### 2.3 向量数据库：ChromaDB

**选择理由**：
- 轻量级、易于 Docker 部署
- Python 原生 SDK，与 FastAPI 集成简单
- 支持余弦距离语义搜索
- 持久化存储，重启不丢失数据

**备选方案**：
| 方案 | 适用场景 |
|------|---------|
| Qdrant | 需要更高性能、支持复杂过滤查询 |
| Milvus | 超大规模（亿级）向量存储 |
| pgvector | 已有 PostgreSQL 基础设施 |
| Pinecone | 不想自己运维，接受 SaaS 方案 |

---

### 2.4 Embedding 模型：paraphrase-multilingual-MiniLM-L12-v2

**特点**：
- 多语言支持（中英文论文标题/摘要均可）
- 模型大小约 418MB，推理速度快
- 基于余弦相似度，适合语义去重
- 本地运行，无 API 调用成本

**向量维度**: 384 维

**去重相似度阈值**: 0.94（经验值）
- 高于 0.94 视为重复（覆盖同一论文的多来源副本、标题措辞差异的同一论文）
- 低于 0.94 视为不同论文

## 3. 数据流详细说明

### 3.1 完整流水线执行步骤

```
Step 1: 多源并发搜索
  ├── arXiv API      → 最新提交的预印本
  ├── PubMed E-utils → 经过同行评审的论文（近 7 天）
  ├── Semantic Scholar → 综合学术数据库，提供引用数
  └── bioRxiv API   → 生命科学预印本（可选）

Step 2: 基础 ID 去重
  └── 检查 paper_id 是否已在 ChromaDB 中存在

Step 3: 语义相似度去重
  ├── 对每篇新论文生成 Embedding
  ├── 查询 ChromaDB 最近邻
  └── 相似度 >= 0.94 则丢弃（视为重复）

Step 4: DeepSeek AI 总结
  ├── 批量处理（batch_size=5）
  ├── 提示词引导输出：核心问题、方法、贡献
  └── 每批间隔 1 秒，避免速率限制

Step 5: 向量存储归档
  ├── 生成 Embedding（标题 + 摘要前 1000 字）
  ├── 写入 ChromaDB（ID + 向量 + 元数据 + 文档）
  └── 支持后续语义检索

Step 6: 多渠道并发推送
  ├── Email (HTML 格式，美观卡片)
  ├── 企业微信/飞书（Markdown 消息，限字数分批）
  ├── Telegram（MarkdownV2 格式，逐篇发送）
  └── GitHub Issues（Markdown 摘要，长期归档）
```

## 4. 关键设计决策

### 4.1 为什么用独立 FastAPI 服务而不是直接在 n8n Code 节点写 Python？

**n8n Code 节点的局限**：
- n8n 的 Code 节点支持 JavaScript，不支持原生 Python
- 即使通过 Execute Command 调用 Python，也无法利用向量 Embedding 的有状态连接
- 第三方 Python 库（chromadb、sentence-transformers）无法在 n8n 沙箱中安装

**FastAPI 服务的优势**：
- 独立进程，可使用任意 Python 库
- ChromaDB 客户端连接池复用，性能更好
- Embedding 模型在服务启动时一次性加载到内存，不用每次重载
- n8n 通过标准 HTTP 调用，天然解耦

### 4.2 去重策略的两层设计

**第一层（精确 ID 去重）**: 速度快，O(n) 查询
**第二层（语义相似度去重）**: 解决以下问题：
- 同一篇论文同时出现在 arXiv 和 Semantic Scholar（不同 ID）
- 标题略有修改的修订版本（arXiv v1 vs v2）
- 预印本和正式发表版本

### 4.3 推送频率设置建议

| 研究方向速度 | 推荐频率 | 每次最多论文数 |
|------------|---------|--------------|
| 热门方向（如 LLM Agency） | 每天 1 次（早 9 点） | 20 篇 |
| 一般方向 | 每天 1 次（早 9 点） | 10 篇 |
| 冷门方向 | 每 3 天 1 次 | 15 篇 |
