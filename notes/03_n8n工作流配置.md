> **记录日期**: 2026-03-09

## 1. 首次登录 n8n

1. 浏览器访问 `http://your-server-ip:5678`
2. 输入 `.env` 中设置的 `N8N_USER` 和 `N8N_PASSWORD`
3. 完成初始账号设置向导

---

## 2. DeepSeek API 配置说明

> ⚠️ **无需在 n8n 中配置 DeepSeek Credential。**

本项目架构中，n8n 仅通过 HTTP Request 调用 `paper-api:8080`，
由 paper-api 容器负责所有 DeepSeek 调用。DeepSeek API Key 只需填写在 `.env` 文件里：

```env
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-chat
```

修改 `.env` 后重启 paper-api 使配置生效：
```bash
cd project && docker compose restart paper-api
```

Settings → **n8n API** 页面（Create an API Key）仅用于外部程序通过 REST 接口控制 n8n，与本项目无关，**无需操作**。

## 4. 工作流节点详细说明

### 节点 1：每日 09:00 触发（Schedule Trigger）

```
配置路径: Trigger Interval → Days
执行时间: 每天 09:00（UTC+8，需确认 TZ=Asia/Shanghai 已设置）
```

**自定义执行时间示例**：
- 每天两次（早9点+晚6点）：添加两个 Schedule Trigger 节点
- 每周一次（周一早9点）：设置 `Weeks`，勾选 `Monday`
- 每隔12小时：设置 `Trigger Interval: Hours`，`Hours Interval: 12`

---

### 节点 2：执行完整流水线（HTTP Request）

```
Method:  POST
URL:     http://paper-api:8080/api/pipeline/run
Timeout: 300000ms（5分钟，AI总结可能较慢）
```

**如果 Paper API 在 n8n 容器外部**（非 Docker Compose 部署）：
```
URL: http://your-server-ip:8080/api/pipeline/run
```

---

### 节点 3：条件判断（If）

判断 `new_papers_count` 是否大于 0，避免没有新论文时发送空消息。

```javascript
// 条件表达式
{{ $json.new_papers_count }} > 0
```

## 5. 高级工作流：分步调用（可选）

如果需要在 n8n 中对每篇论文独立处理（例如：不同关键词推送到不同渠道），
可以使用以下分步工作流替代单一流水线调用：

### 分步工作流架构

```
Schedule Trigger
    │
    ▼
HTTP Request: POST /api/search        ← 获取新论文列表
    │
    ▼
Split In Batches (batchSize: 1)       ← 逐篇处理
    │
    ▼
HTTP Request: POST /api/summarize     ← 对当前论文生成 AI 总结
    │  Body: { "title": "...", "abstract": "..." }
    ▼
Code Node: 格式化消息
    │
    ├──▶ Send Email
    ├──▶ HTTP Request (企业微信 Webhook)
    ├──▶ HTTP Request (飞书 Webhook)
    └──▶ GitHub Node: Create Issue
```

### Code 节点示例（格式化消息）

```javascript
// 获取当前论文和 AI 总结
const paper = $('HTTP Request: /api/search').item.json;
const summary = $input.first().json.summary;

const message = `## 📄 ${paper.title}
**来源**: ${paper.source} | **日期**: ${paper.published_date}
**链接**: ${paper.url}
**作者**: ${paper.authors.slice(0,3).join(', ')}

**🤖 AI 速览**:
${summary}

---`;

return [{ json: { message, title: paper.title, url: paper.url } }];
```

## 6. 配置各推送渠道的 Credential

### 6.1 GitHub Credential

1. **Settings** → **Credentials** → **GitHub**
2. 填入 Personal Access Token（需要 `repo` 权限）
3. 在 GitHub Issues 节点中选择此 Credential

### 6.2 Email (SMTP) Credential

1. **Settings** → **Credentials** → **SMTP**
2. 填写：
   - Host: `smtp.qq.com`（QQ邮箱）
   - Port: `465`（SSL）
   - User: QQ邮箱地址
   - Password: **授权码**（不是登录密码！）

> QQ邮箱获取授权码：登录网页版QQ邮箱 → 设置 → 账户 → IMAP/SMTP服务 → 生成授权码

## 7. 工作流调试技巧

```
1. 使用 "Execute Workflow" 按钮手动触发测试

2. 每个节点执行后，点击节点可查看 Input/Output 数据

3. 如果某节点出错，错误信息会显示在节点右上角的红色感叹号

4. 使用 "Set" 节点注入测试数据，跳过触发器直接测试后续节点

5. 在 Code 节点中使用 console.log() 输出调试信息
   （在节点详情 → Logs 标签页查看）
```

---

## 8. 激活工作流

1. 确认所有节点配置无误，手动触发测试通过后
2. 点击右上角 **Inactive** 开关 → 变为 **Active**（绿色）
3. 工作流会按设定的 Schedule 自动执行

> **提示**: 激活后的工作流会持久化运行，即使 n8n 容器重启后也会恢复（得益于 `--restart unless-stopped`）。
