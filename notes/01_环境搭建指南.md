# 环境搭建指南

> **记录日期**: 2026-03-09
> **系统环境**: Ubuntu 22.04 LTS / 2核4G

---

## 1. 服务器准备

### 1.1 推荐配置

| 配置项 | 最低要求 | 推荐配置 |
|--------|----------|----------|
| CPU | 1核 | 2核+ |
| 内存 | 2GB | 4GB（sentence-transformers 需要约 1GB） |
| 磁盘 | 20GB | 40GB（向量数据库会持续增长） |
| 操作系统 | Ubuntu 20.04 | Ubuntu 22.04 LTS |

### 1.2 安全组/防火墙配置

需要开放以下端口（仅在局域网内访问时可不开放公网）：

```
5678/tcp   — n8n Web 界面
8080/tcp   — Paper API（如需外部访问，否则保持内网）
8000/tcp   — ChromaDB（强烈建议不对公网开放）
```

> **安全提示**: ChromaDB 和 Paper API 建议仅在内网访问。如需外网访问 n8n，务必配置 HTTPS（使用 Nginx 反向代理 + Let's Encrypt）。

## 2. 安装 Docker 与 Docker Compose

### 2.1 Ubuntu 快速安装

```bash
# 更新包索引
sudo apt update && sudo apt upgrade -y

# 安装依赖
sudo apt install -y ca-certificates curl gnupg lsb-release

# 添加 Docker 官方 GPG Key
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
  sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# 添加 Docker 软件源
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装 Docker Engine 和 Docker Compose
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 将当前用户加入 docker 组（避免每次 sudo）
sudo usermod -aG docker $USER
newgrp docker

# 验证安装
docker --version
docker compose version
```

### 2.2 验证结果

```
Docker version 27.x.x, build ...
Docker Compose version v2.x.x
```

## 4. 配置环境变量

```bash
# 从模板创建 .env 文件
cp .env.example .env

# 编辑配置（必填项见下表）
nano .env
```

### 4.1 必填环境变量

| 变量名 | 如何获取 | 示例 |
|--------|---------|------|
| `DEEPSEEK_API_KEY` | [DeepSeek 平台](https://platform.deepseek.com/) → API Keys | `sk-abc123...` |
| `N8N_PASSWORD` | 自定义，不少于 12 位 | `MyStr0ngPwd!` |
| `N8N_ENCRYPTION_KEY` | `openssl rand -hex 32` 生成 | 64位十六进制字符串 |
| `CHROMA_AUTH_TOKEN` | 自定义字符串 | `my-chroma-token-2026` |

### 4.2 生成加密密钥

```bash
# 生成 N8N_ENCRYPTION_KEY（64位十六进制）
openssl rand -hex 32

# 示例输出（请使用你自己生成的）:
# a3f1e2b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2c3d4e5f6a7b8c9d0e1f2
```

## 5. 启动所有服务

```bash
# 一键初始化（会检查 .env、构建镜像、启动服务）
chmod +x scripts/setup.sh
./scripts/setup.sh
```

### 5.1 手动启动（逐步）

```bash
# 仅启动 ChromaDB（先启动，其他服务依赖它）
docker compose up -d chromadb

# 等待 ChromaDB 健康
docker compose ps  # STATUS 列显示 healthy 即可

# 启动 Paper API
docker compose up -d paper-api

# 启动 n8n
docker compose up -d n8n

# 查看所有服务状态
docker compose ps
```

## 7. 常用运维命令

```bash
# 查看所有服务日志
docker compose logs -f

# 查看单个服务日志
docker compose logs -f paper-api
docker compose logs -f n8n

# 重启单个服务
docker compose restart paper-api

# 停止所有服务（不删除数据）
docker compose stop

# 停止并删除容器（保留数据卷）
docker compose down

# 完全清理（包括数据卷，谨慎使用！）
docker compose down -v

# 更新镜像和重建
docker compose pull
docker compose up -d --build
```

## 8. 可选：配置 HTTPS（生产环境推荐）

如果需要通过公网访问 n8n，强烈建议配置 HTTPS：

```bash
# 安装 Nginx 和 Certbot
sudo apt install -y nginx certbot python3-certbot-nginx

# 配置 Nginx 反向代理（文件路径: /etc/nginx/sites-available/n8n）
sudo tee /etc/nginx/sites-available/n8n <<'EOF'
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:5678;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
EOF

sudo ln -s /etc/nginx/sites-available/n8n /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

# 申请 SSL 证书
sudo certbot --nginx -d your-domain.com
```
