## 启用渠道的步骤

1. 在 `project/.env` 中填写对应渠道的 Token/Webhook/密码
2. 在 `project/config/config.yaml` 中将对应渠道的 `enabled` 改为 `true`
3. 重启 Paper API 服务：`docker compose restart paper-api`

```yaml
# config.yaml 中启用渠道
push:
  channels:
    email:        true   # ← 改为 true
    wecom:        false
    feishu:       false
    telegram:     false
    github_issue: false
```

## 渠道 1：Email（邮件推送）

### 获取 SMTP 授权码

#### QQ 邮箱（推荐）

1. 登录 [mail.qq.com](https://mail.qq.com) → 设置（右上角）
2. 点击 **账户** 标签
3. 找到 **IMAP/SMTP服务**，点击 **开启**
4. 使用手机扫描验证后，会生成一个 **16位授权码**（不是登录密码）
5. 将授权码填入 `.env` 的 `EMAIL_SMTP_PASSWORD`

#### Gmail

1. 开启两步验证：Google 账号 → 安全性 → 两步验证
2. 生成应用专用密码：安全性 → 应用专用密码 → 选择"邮件" → 生成
3. SMTP 配置：

```env
EMAIL_SMTP_HOST=smtp.gmail.com
EMAIL_SMTP_PORT=587
EMAIL_SMTP_USER=your@gmail.com
EMAIL_SMTP_PASSWORD=xxxx xxxx xxxx xxxx  # 16位应用密码
```

### 测试 SMTP 连接

```python
import smtplib

with smtplib.SMTP_SSL('smtp.qq.com', 465) as server:
    server.login('your@qq.com', 'your-auth-code')
    print("SMTP 连接成功！")
```

## 渠道 2：企业微信群机器人

### 获取 Webhook URL

1. 打开企业微信 PC 端
2. 进入需要接收推送的**群聊**
3. 点击右上角 **...** → **添加群机器人** → **新创建一个机器人**
4. 填写机器人名称（如"论文助手"），上传头像（可选）
5. 点击**添加** → 复制显示的 **Webhook 地址**

```env
WECOM_WEBHOOK_URL=https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

### 消息效果预览

企业微信机器人支持 Markdown 格式，推送效果如下：

```markdown
## 📚 论文速递 2026年03月09日

**[Cooperative LLM Agents for Complex Task Solving](https://arxiv.org/abs/2603.xxxxx)**
> 📰 arXiv | 📅 2026-03-08

**核心问题**: 现有 LLM Agent 在复杂多步任务中协作效率低...
```

### 限制说明

- 每条 Markdown 消息最大 **4096 字符**（代码已自动分批）
- 每个机器人每分钟最多发送 **20 条**消息
- 不支持 @所有人 等特殊操作

## 渠道 3：飞书群机器人

### 获取 Webhook URL

1. 打开飞书 PC 端，进入目标群聊
2. 点击右上角 **设置** → **群机器人** → **添加机器人**
3. 选择 **自定义机器人**
4. 填写机器人名称，点击**添加**
5. 复制 **Webhook 地址**（可选：设置 IP 白名单和签名校验）

```env
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

### 关于飞书签名校验（可选安全增强）

如果你在飞书机器人设置中启用了**签名校验**，需要修改 `push_service.py` 中的 `FeishuPushChannel.send` 方法，添加 HMAC-SHA256 签名头：

```python
import hashlib
import hmac
import base64
import time

def _sign(self, timestamp: str) -> str:
    """生成飞书 Webhook 签名。"""
    secret = os.getenv("FEISHU_SECRET", "")
    string_to_sign = f"{timestamp}\n{secret}"
    hmac_code = hmac.new(
        string_to_sign.encode("utf-8"),
        digestmod=hashlib.sha256
    ).digest()
    return base64.b64encode(hmac_code).decode("utf-8")

# 在 send() 方法的 payload 中添加:
timestamp = str(int(time.time()))
payload["timestamp"] = timestamp
payload["sign"] = self._sign(timestamp)
```

## 渠道 4：Telegram Bot

### 创建 Bot 并获取 Token

1. 在 Telegram 中搜索 **@BotFather**
2. 发送 `/newbot` 命令
3. 按提示设置 Bot 名称和用户名（用户名必须以 `bot` 结尾）
4. BotFather 会返回 **Bot Token**（格式：`123456789:ABCxxxxxx`）

```env
TELEGRAM_BOT_TOKEN=1234567890:ABCDEFxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 获取 Chat ID

#### 方法 1（个人 Chat ID）

1. 在 Telegram 中搜索 **@userinfobot**
2. 发送任意消息，它会回复你的 Chat ID

#### 方法 2（群组 Chat ID）

1. 将 Bot 添加到目标群组并设为管理员
2. 在群内发送任意消息
3. 访问以下 URL（用你的 Token 替换）:
   ```
   https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates
   ```
4. 在返回的 JSON 中找到 `"chat": {"id": -100xxxxxxxxxx}` （群组 ID 是负数）

```env
TELEGRAM_CHAT_ID=-100xxxxxxxxxx
```

### 消息格式注意事项

Telegram MarkdownV2 格式特殊字符较多，代码中已处理转义。如遇到发送失败，
可在日志中看到具体错误，常见原因：
- 论文标题包含特殊字符 `()[].*` 等未被正确转义

## 渠道 5：GitHub Issues 归档

### 创建专用归档仓库

1. 在 GitHub 创建一个新仓库（如 `paper-archive`），建议设为 **Private**
2. 创建 **Personal Access Token** (PAT)：
   - 访问 GitHub → Settings → Developer Settings → Personal access tokens → Tokens (classic)
   - 点击 **Generate new token**
   - 勾选 `repo` 权限（write access）
   - 生成并复制 Token（只显示一次！）

```env
GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
GITHUB_REPO_OWNER=your_github_username
GITHUB_REPO_NAME=paper-archive
```

### 自动创建 Label

代码会为 Issues 自动打上 `paper-digest` 标签，需要先在仓库中手动创建此 Label：

1. 进入仓库 → **Issues** → **Labels** 标签页
2. 点击 **New label**
3. 名称: `paper-digest`，颜色: `#0075ca`（蓝色）

## 推送效果对比

| 渠道 | 格式 | 实时性 | 搜索能力 | 长期存档 | 适合场景 |
|------|------|--------|---------|---------|---------|
| Email | HTML 富文本 | 较慢 | 邮箱搜索 | ✅ 无限 | 正式记录、详细阅读 |
| 企业微信 | Markdown | 即时 | 群搜索 | ❌ 有限 | 团队协作、快速浏览 |
| 飞书 | 卡片消息 | 即时 | 飞书搜索 | ❌ 有限 | 团队协作、移动端 |
| Telegram | MarkdownV2 | 即时 | 消息搜索 | ❌ 有限 | 个人使用、碎片时间 |
| GitHub Issues | Markdown | 较慢 | GitHub Search | ✅ 永久 | 长期归档、版本控制 |
