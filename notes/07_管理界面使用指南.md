> **记录日期**: 2026-03-10

## 1. Web 管理界面

### 1.1 访问地址

浏览器打开: **http://localhost:8080/admin**

### 1.2 功能概览

管理界面提供三个主要模块：

#### 📊 统计概览
- 实时查看论文总数
- 查看当前使用的向量集合名称
- 查看 Embedding 模型信息

#### 📄 论文库
- **分页浏览**所有已存储的论文（每页 20 篇）
- **实时搜索**：输入关键词过滤论文标题/作者
- **来源筛选**：按 arXiv、PubMed 等来源过滤
- 点击论文标题直接跳转到原文页面
- 显示论文元数据：来源、发表日期、作者、分类
- 显示摘要前 200 字符

#### 🔑 关键词配置
- 查看所有研究方向及其关键词
- 查看每个方向的启用状态（✅ 或 ❌）
- 查看搜索来源和最大结果数配置
- **一键重新加载配置**（修改后无需重启容器）

## 2. 动态修改检索关键词

### 2.1 修改配置文件

编辑 `project/config/keywords.yaml`，添加或修改研究方向：

```yaml
research_areas:
  # 添加新的研究方向
  - name: "Diffusion Models"
    enabled: true
    keywords:
      - "diffusion model"
      - "stable diffusion"
      - "score matching"
      - "denoising diffusion"
    sources:
      - arxiv
    max_results: 10
    arxiv_categories:
      - cs.CV
      - cs.LG
```

### 2.2 热加载配置（无需重启）

**方式1：通过 Web 界面**
1. 修改完 `keywords.yaml` 后保存
2. 打开 http://localhost:8080/admin
3. 切换到"🔑 关键词配置"标签页
4. 点击"🔄 重新加载配置"按钮

**方式2：通过 API**
```bash
curl -X POST http://localhost:8080/api/keywords/reload
```

**方式3：通过 n8n 工作流**
在 n8n 中添加一个 HTTP Request 节点，调用上述 API 端点。

## 3. API 端点说明

### 3.1 关键词管理

```bash
# 获取当前关键词配置
GET http://localhost:8080/api/keywords

# 重新加载关键词配置
POST http://localhost:8080/api/keywords/reload
```

### 3.2 论文查询

```bash
# 列出论文（支持分页和过滤）
GET http://localhost:8080/api/papers/list?limit=20&offset=0&source=arxiv

# 参数说明:
#   limit: 每页数量（1-100，默认 20）
#   offset: 偏移量（默认 0）
#   source: 过滤来源（可选，如 arxiv、pubmed）
```

### 3.3 语义搜索

```bash
# 在论文库中搜索相关论文
POST http://localhost:8080/api/search/semantic
Content-Type: application/json

{
  "query": "multi-agent reinforcement learning",
  "top_k": 10
}
```

## 4. 常见使用场景

### 场景1：添加新的研究方向

1. 编辑 `config/keywords.yaml`，添加新的 `research_area`
2. 在 Web 界面点击"重新加载配置"
3. 在 n8n 中手动触发工作流，测试新关键词是否生效

### 场景2：调整关键词避免重复论文

如果发现推送的论文不够精准，可以：
1. 查看 Web 界面的"📄 论文库"，了解已存储的论文类型
2. 修改 `keywords.yaml`，调整关键词或添加 `arxiv_categories` 过滤
3. 重新加载配置后再次运行

### 场景3：临时禁用某个研究方向

```yaml
- name: "eVTOL Optimization"
  enabled: false  # ← 改为 false 即可禁用
  keywords: ...
```

重新加载配置后，该方向将不会被搜索。

### 场景4：查看历史论文并调整提示词

1. 在 Web 界面浏览已存储的论文
2. 查看 AI 总结质量（通过邮件或 GitHub Issues）
3. 如果总结不够准确，修改 `config/config.yaml` 中的 `prompt_template`
4. 重启 paper-api 容器：`docker compose restart paper-api`

## 5. 注意事项

### ⚠️ 配置文件语法

- 必须使用 **YAML 格式**，注意缩进（2 个空格）
- 字符串包含特殊字符时需要用引号包裹
- 修改后建议在 https://www.yamllint.com/ 验证语法

### ⚠️ 重新加载 vs 重启

| 修改内容 | 生效方式 |
|---------|---------|
| `keywords.yaml` | 重新加载配置即可 |
| `config.yaml` | 需重启 paper-api 容器 |
| `.env` 环境变量 | 需重启整个项目 |

### ⚠️ 向量数据持久化

- 所有论文数据存储在 Docker Volume `chroma_data`
- 即使容器重启，数据也不会丢失
- 如需清空数据库重新测试，执行：
  ```bash
  docker compose down
  docker volume rm autoreadpaper_chroma_data
  docker compose up -d
  ```

## 6. 故障排查

### 问题1：Web 界面打不开

```bash
# 检查容器是否运行
docker compose ps

# 查看日志
docker compose logs -f paper-api

# 重启容器
docker compose restart paper-api
```

### 问题2：修改关键词后不生效

- 确认修改的是 `project/config/keywords.yaml`（不是其他位置）
- 确认在 Web 界面点击了"重新加载配置"
- 在 n8n 中手动触发工作流测试

### 问题3：论文列表显示为空

```bash
# 检查向量数据库状态
curl http://localhost:8080/api/stats

# 如果显示 total_papers: 0，说明还没有存储论文
# 在 n8n 中运行一次完整流水线即可
```
