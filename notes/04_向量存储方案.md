# 向量存储方案

> **记录日期**: 2026-03-09

---

## 1. 为什么需要向量存储？

传统的关键词去重只能匹配相同 ID 或完全相同的标题，无法处理：

| 场景 | 传统去重 | 向量去重 |
|------|---------|---------|
| 同论文出现在 arXiv 和 Semantic Scholar | ❌ 不同 ID，无法去重 | ✅ 语义相同，去重成功 |
| 标题微调的同一论文（arXiv v1 vs v2） | ❌ 无法识别 | ✅ 相似度 >0.94，去重成功 |
| 语义相似但不同的论文 | N/A | ✅ 相似度 <0.94，保留 |
| 积累论文库，支持历史搜索 | ❌ 不支持 | ✅ 向量检索 |

---

## 2. ChromaDB 核心概念

### 2.1 Collection（集合）

类似于关系数据库中的"表"。我们使用一个 Collection `papers` 存储所有论文。

### 2.2 Document（文档）

存储在 ChromaDB 中的原始文本，我们存储 `title + abstract` 拼接后的文本。

### 2.3 Embedding（向量）

文本经过 Embedding 模型编码后的高维向量（384 维），用于语义相似度计算。

### 2.4 Metadata（元数据）

与每个向量关联的结构化字段（标题、来源、日期、URL 等），支持过滤查询。

### 2.5 ID（唯一标识符）

每条记录的唯一 ID，格式为 `{source}:{original_id}`，例如：
- arXiv 论文: `arxiv:2401.12345v2`
- PubMed 论文: `pubmed:38123456`
- Semantic Scholar: `s2:abc123def456`

## 3. Embedding 模型选择

### 3.1 当前使用：paraphrase-multilingual-MiniLM-L12-v2

```
维度:   384
大小:   418 MB
语言:   多语言（支持中英文）
速度:   约 50ms/篇（CPU），5ms/篇（GPU）
```

### 3.2 其他可选模型（在 config.yaml 中修改）

| 模型名 | 维度 | 大小 | 特点 |
|--------|------|------|------|
| `all-MiniLM-L6-v2` | 384 | 91MB | 英文专用，速度最快 |
| `paraphrase-multilingual-MiniLM-L12-v2` | 384 | 418MB | **当前默认，中英文最佳平衡** |
| `all-mpnet-base-v2` | 768 | 438MB | 英文，精度更高 |
| `text2vec-base-chinese` | 768 | 400MB | 中文专用 |

### 3.3 使用 DeepSeek Embedding API（替代方案）

如果不想在服务器上运行 sentence-transformers（节省内存），可以替换为 API 调用：

```python
# 在 VectorStore.__init__ 中修改，使用 OpenAI 兼容的 Embedding API
from openai import AsyncOpenAI

async def embed_via_api(text: str) -> list[float]:
    client = AsyncOpenAI(
        api_key=os.getenv("DEEPSEEK_API_KEY"),
        base_url="https://api.deepseek.com/v1"
    )
    # 注意：截至 2026-03，DeepSeek 尚未开放 embedding endpoint
    # 可使用 OpenAI text-embedding-3-small 替代（约 $0.02/百万 tokens）
    response = await client.embeddings.create(
        model="text-embedding-3-small",
        input=text
    )
    return response.data[0].embedding
```

## 4. 去重阈值调优

### 4.1 阈值的含义

- 余弦相似度 = 1 - 余弦距离
- 阈值 **0.94** 意味着：两篇论文的向量相似度超过 94% 视为重复

### 4.2 调整建议

```yaml
# config.yaml
vector_store:
  dedup_similarity_threshold: 0.94  # 可根据实际情况调整
```

| 阈值 | 效果 |
|------|------|
| > 0.97 | 非常严格，只过滤完全相同的论文 |
| 0.94 (默认) | 平衡，过滤同论文不同来源版本 |
| < 0.90 | 宽松，可能错误丢弃相关但不同的论文 |

**如何测试阈值**：

```bash
# 调用语义搜索 API，查看相似论文之间的分数分布
curl -X POST http://localhost:8080/api/search/semantic \
  -H "Content-Type: application/json" \
  -d '{"query": "large language model agent planning", "top_k": 5}'
```

## 5. 向量库管理

### 5.1 查看存储统计

```bash
curl http://localhost:8080/api/stats
# 返回: {"collection": "papers", "total_papers": 1234, "embedding_model": "..."}
```

### 5.2 直接操作 ChromaDB

```python
import chromadb

client = chromadb.HttpClient(
    host="localhost", port=8000,
    settings=chromadb.config.Settings(
        chroma_client_auth_provider="chromadb.auth.token_authn.TokenAuthClientProvider",
        chroma_client_auth_credentials="your-chroma-token"
    )
)

collection = client.get_collection("papers")

# 查看总数
print(collection.count())

# 按来源过滤查询
results = collection.get(
    where={"source": "arXiv"},
    limit=10,
    include=["metadatas"]
)
print(results["metadatas"])
```

### 5.3 数据备份

ChromaDB 数据存储在 Docker Volume `chroma_data` 中：

```bash
# 备份到本地目录
docker run --rm \
  -v autoreadpaper_chroma_data:/source \
  -v $(pwd)/backup:/backup \
  alpine tar czf /backup/chroma_backup_$(date +%Y%m%d).tar.gz -C /source .
```
